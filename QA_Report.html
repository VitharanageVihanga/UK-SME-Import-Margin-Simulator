Project Overview

The UK SME Import Margin Simulator is a data-driven decision-support tool that combines HMRC trade statistics (84 monthly data files, 2018–2024) with ONS foreign trade data to provide real-world margin simulation, FX risk analysis, and predictive forecasting for UK importers.

Testing Scope & Environment

In Scope

  • All pure-function modules in scripts/ — 14 Python files, 1,033 executable statements

  • Analytics: forecasting engine, trend analysis, advanced risk metrics, backtesting

Out of Scope

  • ONS pipeline scripts — offline data-preparation scripts run once during project setup
  • Performance load testing — beyond measuring test suite execution time

Test Strategy

MethodologyApplied ToTechniques
Unit Testing All 14 scripts/ modules Equivalence partitioning, boundary value analysis, decision coverage
Boundary Value Analysis risk_label.py, margin_model.py, confidence_band.py Exact threshold values (0%, 5%, 10%), off-by-one checks
Equivalence Partitioning All classification functions Valid / invalid input classes, coverage tier classes (High/Partial/Low/None)
Error Path Testing All modules with graceful degradation None inputs, empty DataFrames, missing columns, invalid strings, insufficient data
Integration Testing Data pipeline, forecast report, stress test runner Fixture chaining, monkeypatching of I/O layers
Regression Testing All 330 tests (re-run post-cleanup) Full re-execution after artefact removal
Mathematical Validation VaR, margin calculation, scenario runner, growth rates Closed-form expected values, monotonicity assertions, Cauchy-Schwarz (MAE ≤ RMSE)

Design Principles

  • Determinism — every test using randomness is seeded with a fixed integer
  • No external I/O — file reads use pytest tmp_path or monkeypatching; no live API calls
  • Tolerance-aware assertions — floating-point comparisons use pytest.approx() with explicit absolute tolerance
  • Guard assertions — probabilistic branches assert structure, not exact values, to avoid flakiness across statsmodels versions

Why the ETL Scripts Show 0% Coverage

The four P4 scripts are offline data-preparation tools run once during initial project setup to process raw ONS Excel files into cleaned CSVs. They are not invoked at runtime by app.py, and their outputs are already present in data/processed/. Excluding them from unit testing is architecturally sound and industry-standard practice for ETL pipeline scripts.
